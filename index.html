How It Works

Sync algorithm, simplified

Each sync run follows a deterministic pipeline that guarantees consistency.

Fetch & Lock

Acquire a file lock, load state, then fetch tasks from both Google Tasks and Microsoft To Do APIs.

Match & Diff

Cold-start match on first run. For known tasks, diff every field against the last canonical snapshot.

Resolve Conflicts

If both providers changed the same field, last-write-wins by timestamp. All conflicts are logged for auditing.

Push & Save

Fan out the resolved canonical state to all providers. Persist updated mappings, tombstones, and watermarks.

Architecture

Clean, extensible, provider-agnostic

The sync engine doesn't know about Google or Microsoft. Providers implement a simple TaskProvider interface â€” adding new ones is straightforward.

Sync Modes

Choose how your data flows

Three modes to match your workflow. Set once, forget about it.

⇄

Bidirectional

Changes flow both ways. The default and most common mode â€” full two-way sync.
